## 1. Executive Summary

### 핵심 발견

| 지표 | Vertical SaaS | Horizontal SaaS |
| --- | --- | --- |
| **성장률** | 31% | 28% |
| **NRR (순수익 유지율)** | 120%+ | 100-110% |
| **이탈률** | 낮음 (전문성) | 높음 (전환 용이) |
| **시장 침투율** | 50%+ (Shopify 29%, Mindbody 61.5%) | 5-10% |
| **고객 획득 비용** | 낮음 (타겟 마케팅) | 높음 (광범위 마케팅) |

### 시장 규모

- **2025년 Vertical SaaS 시장**: 1,574억 달러
- **CAGR**: 23.9%
- **2030년 전망**: 2,500억 달러+

### 핵심 인사이트

```
"Horizontal SaaS는 망치를 만든다. 
Vertical SaaS는 특정 지붕용 손톱을 정확히 박는 
도구를 만든다."
```

**3대 성공 요인**:

1. **깊은 산업 이해** - 워크플로우, 규제, 언어 내재화
2. **Land & Expand 전략** - 핵심 워크플로우 → 전체 플랫폼
3. **Embedded Fintech** - 결제, 대출, 급여 통합으로 수익 극대화

## 2. Vertical SaaS 개요

### 2.1 정의와 특징

| 구분 | Horizontal SaaS | Vertical SaaS |
| --- | --- | --- |
| **타겟** | 모든 산업 | 특정 산업 |
| **예시** | Salesforce, Slack, Zoom | Veeva, Toast, Procore |
| **커스터마이징** | 높은 필요 (비용↑) | 거의 없음 (즉시 사용) |
| **구현 기간** | 3-6개월 | 1-2주 |
| **고객 성공** | 일반적 | 산업 전문가 배정 |

### 2.2 왜 Vertical SaaS가 뜨는가?

#### 1. 레거시 산업의 디지털 전환 가속

- 건설, 농업, 제조업 등 전통 산업의 SaaS 도입 급증
- COVID-19 이후 필수 인프라로 인식

#### 2. 높은 진입장벽 = 높은 수익성

- 산업 특화 지식 필요
- 규제 준수 요건
- 높은 전환 비용 → 낮은 이탈률

#### 3. Embedded Fintech 기회

- 산업 내 금융 흐름 포착
- Toast: SaaS 매출의 15%, 핀테크 매출의 85%

## 3. 성공 사례 심층 분석

### 3.1 Toast - 레스토랑 운영체제

#### 기업 개요

| 항목 | 내용 |
| --- | --- |
| **설립** | 2011년 |
| **IPO** | 2021년 |
| **현재 가치** | 87억 달러 |
| **2024년 매출** | 50억+ 달러 |
| **고객 수** | 11.3만+ 레스토랑 |
| **NRR** | 121% (2020-2023) |

#### 성공 전략

**1. 웨지 전략 (Wedge Strategy)**

```
2011년: POS (포스) 시스템 시작
    ↓
2015년: 온라인 주문, 예약 추가
    ↓
2018년: 급여, 인사 관리 추가
    ↓
2020년: 핀테크 (대출, 결제) 본격화
    ↓
2024년: 레스토랑 운영체제 완성
```

**2. COVID-19 위기를 기회로**

- 비대면 결제 (Contactless Payment)
- 커브사이드 픽업 알림
- 플랫 배달료 구조
- **속도**: 고객 피드백 → 기능 출시 = 2주

**3. 수익 모델 혁신**

| 수익원 | 비중 | 특징 |
| --- | --- | --- |
| **구독** | 15% | SaaS 기본 요금 |
| **결제 수수료** | 70%+ | 거래당 수수료 |
| **대출 (Toast Capital)** | 10% | 현금 흐름 기반 대출 |
| **기타 핀테크** | 5% | 급여, 보험 등 |

**핵심 통찰**:

> "우리는 SaaS 회사가 아니다. 핀테크 회사이며, SaaS는 유통 채널일 뿐이다." — Toast 경영진

#### Toast의 고객 확장 전략

| 단계 | 전략 | 결과 |
| --- | --- | --- |
| **Land** | 2개 이하 매장 소형 레스토랑 타겟 | 빠른 영업 사이클 |
| **Expand** | 다중 매장 체인으로 확장 | ARPU 3-5배 증가 |
| **Platform** | 전국 체인 + 프랜차이즈 | 장기 계약, 높은 LTV |

## 5. 수익 모델 혁신: Embedded Fintech

### 5.1 개요

**Embedded Fintech** = Vertical SaaS + 금융 서비스 통합

| 구성 요소 | 설명 | 예시 |
| --- | --- | --- |
| **결제** | 플랫폼 내 직접 결제 | Toast Payments |
| **대출** | 현금 흐름 기반 대출 | Toast Capital |
| **급여** | 자동 급여 처리 | Gusto (Toast 파트너) |
| **보험** | 산업 특화 보험 | Next Insurance |
| **은행** | 비즈니스 계좌 | Shopify Balance |

### 5.2 수익 구조 변화

#### 전통적 SaaS

```
수익 = 구독료 × 고객 수
     = $100/월 × 1,000명
     = $100,000/월
```

#### Embedded Fintech SaaS

```
수익 = 구독료 + (거래액 × 수수료) + 금융 수익
     = $50/월 × 1,000명 + ($1M × 2.9%) + $20,000
     = $50,000 + $29,000 + $20,000
     = $99,000/월
     
→ 구독 비중: 50% → 핀테크 비중: 50%
→ ARPU 2배 증가
```

### 5.3 성공 사례 비교

| 기업 | SaaS 매출 | 핀테크 매출 | 핀테크 비중 |
| --- | --- | --- | --- |
| **Toast** | 7.06억 달러 | 41억 달러 | **85%** |
| **Shopify** | 13억 달러 | 53억 달러 | **80%** |
| **Mindbody** | 1.5억 달러 | 1.8억 달러 | **55%** |
| **ServiceTitan** | 6억 달러 | 3억 달러 | **33%** |

### 5.4 Embedded Fintech 구현 단계

```
Phase 1: 결제 통합 (Payments)
    └── 가장 쉬운 시작점
    └── 고객 행동 데이터 확보
    
Phase 2: 대출/자금 조달 (Lending)
    └── "결제는 웨지, 대출은 지갑"
    └── 현금 흐름 데이터로 리스크 평가
    
Phase 3: 급여/보험 (Payroll/Insurance)
    └── 완전한 금융 인프라
    └── 고객 이탈 방지
```

## 6. 성장 전략: Land & Expand

### 6.1 전략 개요

```
┌─────────────────────────────────────────────────────────────┐
│                    Land & Expand 전략                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   LAND (진입)                                               │
│   └── 핵심 워크플로우 1개를 완벽하게 해결                   │
│   └── 낮은 마찰, 빠른 가치 인식                             │
│   └── 예: Toast의 POS, Veeva의 CRM                          │
│                                                             │
│        ↓                                                    │
│                                                             │
│   EXPAND (확장)                                             │
│   └── 인접 워크플로우 추가                                  │
│   └── 데이터/신뢰 기반 자연스러운 확장                      │
│   └── 예: POS → 급여 → 대출 → 마케팅                        │
│                                                             │
│        ↓                                                    │
│                                                             │
│   PLATFORM (플랫폼)                                         │
│   └── 산업 운영체제가 됨                                    │
│   └── 타사 통합/마켓플레이스                                │
│   └── 예: Toast OS, Procore Platform                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 핵심 지표: NRR (Net Revenue Retention)

| NRR 수준 | 의미 | 예시 기업 |
| --- | --- | --- |
| **100%** | 유지 | 평균 SaaS |
| **110%** | 성장 | 좋은 SaaS |
| **120%** | 강한 성장 | Vertical SaaS 평균 |
| **130%+** | 탁월 | Shopify (125%), Veeva (124%) |

### 6.3 고객 확장 전략

| 단계 | 전략 | 목표 |
| --- | --- | --- |
| **1. 웨지 제품** | 가장 아픈 문제 해결 | 빠른 도입 |
| **2. 사용자 성공** | 온보딩 및 교육 | 높은 활성화 |
| **3. 기능 확장** | 인접 워크플로우 | ARPU 증가 |
| **4. 조직 확장** | 부서/지점 추가 | 로고 확장 |
| **5. 플랫폼화** | 생태계 구축 | 높은 전환 비용 |

## 결론

### 핵심 메시지

1. **Vertical SaaS는 "깊이"의 게임**

   - 한 산업을 완벽히 이해하고 장악하라
   - Horizontal과 경쟁하지 말고 레거시와 경쟁하라

2. **Embedded Fintech은 선택이 아닌 필수**

   - 순수 SaaS만으로는 성장 한계
   - 산업의 금융 흐름을 포착하라

3. **AI는 기회이자 위협**

   - Vertical AI가 새로운 유니콘을 만들 것
   - 기존 Vertical SaaS도 AI 통합 필수

4. **속도보다 지속 가능성**

   - 빠른 성장보다 높은 NRR
   - 고객 성공이 최고의 성장 전략

### 마지막 조언

> "Horizontal SaaS는 이미 끝났다. 미래는 Vertical SaaS에 있고, 그 다음은 Vertical AI에 있다."
