# 코딩 핸드오프 문서 (CODING_HANDOFF)

> **이 문서의 규칙**
> - 한 handoff는 한 사이클과 요구사항 1~3개만 다룬다.
> - `요구사항 ID`, `영향 범위`, `금지 변경`, `계약` 중 하나라도 비어 있거나 서로 충돌하면 AI 코딩을 시작하지 않는다.
> - AI는 빈칸을 추측해서 채우지 않는다. 누락, 충돌, 확인 불가 항목은 중단 사유로 기록한다.
> - 검증 명령은 실행 명령, 기대 결과, 통과 기준을 모두 포함해야 하며, 모두 통과해야 한다.
> - placeholder가 남아 있으면 AI 코딩을 시작할 수 없고, 보안 체크는 생략할 수 없으며 해당 없음은 이유를 쓴다.

## 하지 않을 것 (선택)
- Kimi CLI 호출 방식 변경: 외부 의존 검증은 이번 로컬 노트 저장 계약 범위 밖이다.
- 뉴스 수집 소스 변경: REQ-101은 저장 경로 안전화만 다룬다.
- baseline-frozen-v4 파일 수정: 동결 기준선 변경은 별도 REMAND 안건이다.

해당 없음 — 데이터베이스, 외부 설정 저장소, 운영 데이터 마이그레이션이 없는 로컬 스크립트 설정 변경이다. 기존 config.ini에는 [저장] notes_dir 기본값을 추가하며, 값이 없으면 안전한 기본 동작을 유지한다.

## 리뷰 포인트 (선택)
- --no-notes가 파일 복사를 실제로 0회로 만드는지 확인한다.
- notes_dir 생성/쓰기 실패가 전체 스크립트 크래시로 전파되지 않는지 확인한다.
- Kimi 호출과 뉴스 생성 경로가 변경되지 않았는지 확인한다.
