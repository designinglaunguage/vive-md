## 1. Google Material Design

### 개요
- **개발사:** Google
- **버전:** Material Design 3 (M3)
- **웹사이트:** https://m3.material.io/
- **GitHub:** https://github.com/material-components

### 핵심 원칙
| 원칙 | 설명 |
|------|------|
| **Material as a Metaphor** | 종이와 잉크에서 영감을 받은 시각적 언어 |
| **Bold, Graphic, Intentional** | 의도적인 색상, 이미지, 타이포그래피 사용 |
| **Motion Provides Meaning** | 의미를 전달하는 모션과 애니메이션 |

### 주요 구성 요소
- **Action:** 버튼, 플로팅 액션 버튼, 아이콘 버튼
- **Containment:** 카드, 칩, 다이얼로그, 사이드 시트
- **Communication:** 배너, 배지, 프로그레스 인디케이터, 스낵바
- **Navigation:** 바텀 네비게이션, 날짜 피커, 네비게이션 드로어, 탭
- **Selection:** 체크박스, 메뉴, 라디오 버튼, 슬라이더, 스위치
- **Text Input:** 텍스트 필드

### 특징
- 오픈소스 코드 기반
- 크로스 플랫폼 지원 (Android, iOS, Web, Flutter)
- 동적 컬러 시스템 (Material You)
- 46개 연구, 18,000+ 참가자의 UX 리서치 기반

## 2. Apple Human Interface Guidelines

### 개요
- **개발사:** Apple
- **웹사이트:** https://developer.apple.com/design/human-interface-guidelines
- **플랫폼:** iOS, macOS, watchOS, tvOS, visionOS

### 핵심 원칙
| 원칙 | 설명 |
|------|------|
| **Hierarchy** | 명확한 시각적 계층 구조 |
| **Harmony** | 하드웨어와 소프트웨어의 조화 |
| **Consistency** | 플랫폼 전반의 일관된 디자인 |
| **Clarity** | 명확성 - 모든 크기에서 가독성 |
| **Deference** | 콘텐츠에 대한 존중 |
| **Depth** | 시각적 레이어와 리얼리즘 |

### 주요 구성 요소
- **Bars:** 날비게이션 바, 탭 바, 툴 바, 서치 바
- **Views:** 테이블 뷰, 컬렉션 뷰, 스플릿 뷰
- **Controls:** 버튼, 스위치, 슬라이더, 세그먼트 컨트롤
- **System Experiences:** 위젯, 라이브 액티비티, 알림

### 특징
- Liquid Glass 디자인 (최신)
- SF Symbols 아이콘 시스템
- San Francisco & New York 커스텀 폰트
- 접근성 중심 설계

## 3. Microsoft Fluent Design System

### 개요
- **개발사:** Microsoft
- **버전:** Fluent 2
- **웹사이트:** https://fluent2.microsoft.design/
- **GitHub:** https://github.com/microsoft/fluentui

### 핵심 원칙
| 원칙 | 설명 |
|------|------|
| **Adaptive** | 다양한 디바이스와 입력 방식에 적응 |
| **Emotional** | 감성적 연결을 위한 디자인 |
| **Intuitive** | 직관적인 상호작용 |

### 주요 구성 요소
- **Web:** Avatar, Checkbox, Combobox, Divider, Drawer, Menu, Radio, Slider, Tabs, Tooltip
- **iOS/Android:** 네이티브 모바일 컴포넌트
- **Windows:** 네이티브 Windows 컴포넌트

### 특징
- Web Components 기반 (프레임워크 독립적)
- React, Vue, Angular, .NET/Blazor 지원
- Microsoft 365 앱에서 사용 (Word, Excel, Teams, PowerBI)
- 접근성 도구 내장 (A11y Color Contrast Checker, Focus Order)

## 4. IBM Carbon Design System

- **GitHub:** https://github.com/carbon-design-system/carbon

|------|------|

### 특징

- React, Angular, Vue, Web Components 지원
- Monorepo 아키텍처 (Yarn workspaces + Lerna)
- BEM 네이밍 규칙

## 6. Airbnb Design Language System (DLS)

### 개요
- **개발사:** Airbnb
- **웹사이트:** https://airbnb.design/

### 핵심 원칙
| 원칙 | 설명 |
|------|------|
| **Unified** | 모든 요소가 더 큰 전체의 일부 |
| **Universal** | 전 세계 사용자를 위한 포용적 디자인 |
| **Iconic** | 명확하고 집중적인 디자인 |
| **Conversational** | 모션을 통한 생동감 있는 소통 |
| **Design for Belonging** | 소속감을 주는 디자인 |
| **Every Detail Matters** | 디테일이 스토리텔링 |

### 특징
- Design Language System (DLS)으로 내부적으로 사용
- Unified, Not Uniform 철학
- 감성적 디자인 중심
- 접근성, 현지화, 문화적 뉘앙스 반영
- 디자인 시스템 관리를 위한 엄격한 프로세스

### 특징
- AI 디자인 가이드라인

## 10. Uber Base Design System

### 개요
- **개발사:** Uber
- **웹사이트:** https://base.uber.com/

### 핵심 원칙
| 원칙 | 설명 |
|------|------|
| **Unified Experience** | Uber 생태계 전반의 통합된 경험 |
| **Flexible** | 혁신을 위한 유연성 |
| **Scalable** | 확장 가능한 디자인 시스템 |

### 주요 구성 요소
- **Buttons:** 다양한 스타일과 크기의 버튼
- **Input and Selection:** 입력 필드, 선택 컴포넌트
- **Indicator and Status:** 상태 표시 인디케이터
- **Content:** 콘텐츠 표시 컴포넌트

### 특징
- Uber의 모든 제품과 서비스에 적용
- 디자이너와 엔지니어를 위한 중앙화된 라이브러리
- 일관된 브랜드 아이덴티티 유지
- 접근성 및 사용성 가이드라인

## 11. GitHub Primer

### 개요
- **개발사:** GitHub
- **웹사이트:** https://primer.style/
- **GitHub:** https://github.com/primer

### 주요 구성 요소
- **Actions:** 버튼, 아이콘 버튼, 액션 바, 액션 리스트, 액션 메뉴
- **Forms:** 체크박스, 라디오, 텍스트 입력, 텍스트 영역, 셀렉트, 토글 스위치
- **Layout:** 페이지 레이아웃, 스택, 스플릿 페이지 레이아웃
- **Navigation:** 브레드크럼, 내비게이션 리스트, 페이지네이션, 언더라인 내비
- **Data Display:** 아바타, 아바타 스택, 카운터 라벨, 라벨, 상태 라벨, 트리 뷰, 데이터 테이블
- **Feedback:** 배너, 인라인 메시지, 토스트, 툴팁, 스피너, 프로그레스 바
- **Overlays:** 다이얼로그, 모달, 팝오버, 오버레이

### 특징
- Primer Product UI, Primer Brand UI, Primer Marketing UI로 구분
- Octicons 아이콘 시스템
- Primitives 디자인 토큰
- React 및 Ruby ViewComponents 제공
- 2026년 2월부터 ViewComponents는 유지보수 모드
